# GPU 基数排序 (Radix Sort)
## 基数排序 Radix Sort
### 概念
基数排序 (Radix Sort) 是一种非比较型的整数排序算法。它的核心思想不是通过不断地比较两个数字的大小来决定位置，而是将整数按位切割成不同的数字，然后按每个位数分别进行分配和收集。
> 在 GPU（图形处理器）上，基数排序是最常用、效率最高的排序算法之一（例如 NVIDIA 的 Thrust 库底层的大规模排序就是基于 Radix Sort）。
### 代码实现
```cpp
// 基数排序代码
// 时间复杂度：O(nd)  空间复杂度:O(n)  稳定性：稳定的排序
void RadixSort(int arr[], int size)
{
	// 寻找最大值确定位数
    int maxData = arr[0];
	for (int i = 1; i < size; i++)
	{
		if (maxData < abs(arr[i]))
		{
			maxData = abs(arr[i]);
		}
	}

	int len = to_string(maxData).size();
    // 构造 bucket
	vector<vector<int>> vecs;
	// 构造取基数的算子
    int mod = 10; // 取模算子
	int dev = 1; // 取位算子

	for (int i = 0; i < len; mod *= 10, dev *= 10, i++) // O(d) d：数据的长度
	{
		vecs.resize(20); // 20个桶，为了能处理负数 -9 - 9

		for (int j = 0; j < size; j++) // O(n)
		{
			// 得到当前元素第i个位置的数字
			int index = arr[j] % mod / dev + 10;
			vecs[index].push_back(arr[j]);
		}

		// 依次遍历所有的桶，把元素拷贝回原始的数组当中
		int idx = 0;
		for (auto vec : vecs) // O(20) 
		{
			for (int v : vec) // O(n)    O(20)*O(n) = O(n)
			{
				arr[idx++] = v;
			}
		}

		vecs.clear();
	}
}
```
### 为什么GPU更倾向于使用基数排序而非其他（如快排、归并等）
* **在 CPU 上**，由于单核性能强、拥有复杂的分支预测机制，快速排序的递归和 ```if (a > b)``` 运行得飞快。
* **但在 GPU 上**，成百上千个轻量级核心（thread）是以组（Warp）为单位步调一致地执行同一条指令的。如果代码里有 ```if...else```，导致部分线程走 ```if```，部分走 ```else```，GPU 只能让它们串行执行，这被称为 Warp Divergence，会彻底摧毁并行性能。
  
## GPU 如何实现 Radix Sort 
> 在普通的 CPU 基数排序中，我们通常按<u>**十进制（0-9）**</u>分 10 个桶。但在 GPU 上，为了极致的硬件效率，通常按<u>**二进制位（Radix-2）**</u>或者<u>**多个二进制位（如 Radix-16，即 4 bits）**</u>来进行拆分。

假设我们按最简单的 Radix-2（只分 0 和 1 两个桶）从低位到高位遍历所有比特位。对于每一个比特位，GPU 会极其高效地执行以下**三个并行核心步骤**：
### 1. Histogram / Counting（并行直方图统计）
GPU 将整个大数组切分成无数个小块（Block），分配给不同的线程组。每个线程组独立统计自己负责的区块中，当前比特位是 0 的有几个，是 1 的有几个。
### 2. Prefix Sum / Scan（并行前缀和 —— 灵魂步骤）
   我们现在知道了每个小块里 0 和 1 的数量，但还要知道它们在最终数组里的具体存放索引（位置）。
   前缀和（Prefix Sum）算法可以并行地累加这些统计数据。例如，如果前面的区块总共有 150 个 0，那么当前区块的第一个 0 就应该被放到输出数组的第 151 个位置。
   前缀和算法（如 Blelloch Scan）利用树状归约结构，在 GPU 上可以在 $O(\log N)$ 步内并行完成，这是 GPU 排序的性能引擎。
### 3. Scatter（并行数据分散 / 写入）
   现在，所有的线程都知道了自己手中的数据应该去哪里。成千上万个线程同时将原数组中的数据直接写入到新数组的精确位置中。

执行完这三步，当前位的排序就结束了。接着对下一个更高的比特位重复此过程，直到最高位处理完毕，整个数组就神奇地变有序了。
